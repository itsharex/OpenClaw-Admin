## 审计发现

### 🔴 高风险 (High)

#### 1. 认证凭据硬编码风险
**位置**: `server/index.js:372`  
**问题**: 使用环境变量存储的认证凭据在登录验证时直接比较明文密码
```javascript
if (username !== envConfig.AUTH_USERNAME || password !== envConfig.AUTH_PASSWORD) {
```
**影响**: 如果.env 文件被泄露，攻击者可直接获取管理员凭据  
**建议**: 
- 使用密码哈希存储和验证
- 实现多因素认证
- 添加登录失败速率限制（已有基础实现但需加强）

#### 2. 系统命令执行风险
**位置**: `server/index.js:578-579`, `server/index.js:1369`  
**问题**: 
- npm 更新功能直接执行 `execSync('npm install -g ${packageSpec}')`
- PTY 终端会话允许执行任意 shell 命令
**影响**: 如果权限控制被绕过，攻击者可执行任意系统命令  
**建议**:
- 对所有命令参数进行严格白名单验证
- 限制可执行命令的范围
- 添加命令执行审计日志
- 考虑使用沙箱环境

#### 3. VNC 远程桌面密码处理
**位置**: `server/index.js:1553`, `server/index.js:1759`  
**问题**: VNC 密码以明文形式传递和存储，且使用 `-passwd` 参数暴露在进程列表中
**影响**: 本地用户可通过 `ps` 命令查看 VNC 密码  
**建议**:
- 使用 VNC 密码文件（.vnc/passwd）而非命令行参数
- 实现加密的会话管理

---

### 🟠 中风险 (Medium)

#### 4. CORS 配置过于宽松
**位置**: `server/index.js:85-92`  
**问题**: CORS 配置允许所有来源访问
```javascript
app.use(cors({
  origin: true,
  credentials: true,
  ...
}))
```
**影响**: 任何网站都可跨域访问 API，可能导致 CSRF 攻击  
**建议**:
- 明确指定允许的源域名
- 根据环境动态配置 CORS 策略

#### 5. 文件路径遍历防护依赖单一检查
**位置**: `server/index.js:754-777`  
**问题**: `safePath` 函数使用字符串前缀检查防止路径遍历，但可能受符号链接攻击
**影响**: 如果攻击者能创建符号链接，可能访问限制外的文件  
**建议**:
- 使用 `fs.realpath` 解析符号链接后再次验证
- 添加文件访问审计日志

#### 6. SQL 注入防护不足
**位置**: `server/index.js:2219-2508`  
**问题**: 虽然使用了参数化查询，但表名和字段名无法参数化
**影响**: 如果未来添加动态表名/字段名查询，可能存在 SQL 注入风险  
**建议**:
- 对动态表名/字段名使用白名单验证
- 定期审查所有数据库查询

#### 7. 会话管理
**位置**: `server/index.js:405`, `server/auth.js:207`  
**问题**: 
- Token 使用 UUID 生成，但存储方式简单
- 会话过期检查依赖内存 Map，重启后失效
**影响**: 服务器重启后所有会话失效，用户体验差  
**建议**:
- 将会话存储到数据库
- 实现会话持久化

---

### 🟡 低风险 (Low)

#### 8. 缺少安全 HTTP 头
**位置**: `server/index.js`  
**问题**: 缺少以下安全头：
- Content-Security-Policy
- X-Content-Type-Options
- X-Frame-Options
- X-XSS-Protection  
**建议**: 添加完整的安全头配置

#### 9. 错误信息泄露
**位置**: 多处 API 端点  
**问题**: 部分错误响应包含详细的技术信息  
**建议**:
- 生产环境隐藏详细错误信息
- 使用通用错误消息

#### 10. 日志敏感信息
**位置**: `server/index.js:377`, `server/index.js:393`  
**问题**: 审计日志可能包含敏感信息  
**建议**:
- 对日志中的敏感字段进行脱敏
- 实现日志访问控制
